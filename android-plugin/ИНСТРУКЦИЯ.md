# Сборка APK Studyfay с оплатой RuStore

## Что понадобится

- Компьютер на Windows, Mac или Linux
- Android Studio (скачать: https://developer.android.com/studio)
- Node.js 18+ (скачать: https://nodejs.org)

---

## Шаг 1. Скачай проект

В poehali.dev нажми **Скачать → Скачать код** — скачается ZIP-архив.

Распакуй его в любую папку, например `C:\Studyfay` (Windows) или `~/Studyfay` (Mac/Linux).

---

## Шаг 2. Установи зависимости

Открой терминал (командную строку) в папке проекта и выполни:

```bash
npm install
```

Подожди пока установится (1-3 минуты).

---

## Шаг 3. Собери веб-часть

```bash
npm run build
```

Появится папка `dist/` — это собранный сайт, который будет внутри APK.

---

## Шаг 4. Добавь Android-проект

```bash
npx cap add android
```

Если уже есть папка `android/`, пропусти этот шаг и выполни:

```bash
npx cap sync
```

---

## Шаг 5. Открой проект в Android Studio

```bash
npx cap open android
```

Или вручную: открой Android Studio → File → Open → выбери папку `android/` внутри проекта.

Android Studio начнёт скачивать Gradle и зависимости — подожди пока внизу исчезнет прогресс-бар (3-10 минут).

---

## Шаг 6. Добавь RuStore SDK

### 6.1. Открой файл `android/settings.gradle`

Найди блок `repositories` и добавь строку с RuStore:

```gradle
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.DEFAULT)
    repositories {
        google()
        mavenCentral()
        maven { url = uri("https://artifactory-external.vkpartner.ru/artifactory/maven") }
    }
}
```

### 6.2. Открой файл `android/build.gradle` (корневой, НЕ app/)

Найди блок `plugins` и добавь Kotlin:

```gradle
plugins {
    id 'com.android.application' version '8.2.0' apply false
    id 'org.jetbrains.kotlin.android' version '1.9.22' apply false
}
```

### 6.3. Открой файл `android/app/build.gradle`

**В самом начале** файла добавь плагин Kotlin:

```gradle
plugins {
    id 'com.android.application'
    id 'org.jetbrains.kotlin.android'
}
```

**В блоке `android {`** добавь:

```gradle
android {
    // ... существующие настройки ...
    
    kotlinOptions {
        jvmTarget = '1.8'
    }
}
```

**В блоке `dependencies {`** добавь:

```gradle
dependencies {
    // ... существующие зависимости ...
    
    implementation(platform("ru.rustore.sdk:bom:2025.11.01"))
    implementation("ru.rustore.sdk:pay")
    implementation("androidx.lifecycle:lifecycle-runtime-ktx:2.7.0")
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-android:1.7.3")
}
```

## Шаг 7. Скопируй файлы плагина оплаты

Из папки `android-plugin/` скопируй 3 файла:

| Что копировать | Куда вставить |
|---|---|
| `MainActivity.kt` | `android/app/src/main/java/ru/studyfay/app/` |
| `RuStoreBillingPlugin.kt` | `android/app/src/main/java/ru/studyfay/app/` |
| `AndroidManifest.xml` | `android/app/src/main/` (заменить существующий) |

**Важно:** Удали старые Java-файлы если они есть:
- `android/app/src/main/java/ru/studyfay/app/MainActivity.java` — УДАЛИТЬ
- Файлы `.kt` заменяют `.java`

Команды для терминала (из корня проекта):

**Windows (PowerShell):**
```powershell
Remove-Item -Force "android\app\src\main\java\ru\studyfay\app\MainActivity.java" -ErrorAction SilentlyContinue
Copy-Item "android-plugin\MainActivity.kt" "android\app\src\main\java\ru\studyfay\app\"
Copy-Item "android-plugin\RuStoreBillingPlugin.kt" "android\app\src\main\java\ru\studyfay\app\"
Copy-Item "android-plugin\AndroidManifest.xml" "android\app\src\main\AndroidManifest.xml"
```

**Mac/Linux:**
```bash
rm -f android/app/src/main/java/ru/studyfay/app/MainActivity.java
cp android-plugin/MainActivity.kt android/app/src/main/java/ru/studyfay/app/
cp android-plugin/RuStoreBillingPlugin.kt android/app/src/main/java/ru/studyfay/app/
cp android-plugin/AndroidManifest.xml android/app/src/main/AndroidManifest.xml
```

## Шаг 8. Синхронизируй в Android Studio

В Android Studio нажми **File → Sync Project with Gradle Files** (или кнопку со слоником вверху).

Если появятся ошибки — прочитай текст ошибки. Чаще всего:
- «SDK not found» → установи Android SDK 34 через SDK Manager
- «Kotlin not found» → проверь что шаг 6.2 выполнен правильно

---

## Шаг 9. Собери APK

### Для тестирования (debug):

В Android Studio: **Build → Build Bundle(s) / APK(s) → Build APK(s)**

APK появится в: `android/app/build/outputs/apk/debug/app-debug.apk`

### Для публикации (release):

В Android Studio: **Build → Generate Signed Bundle / APK**

1. Выбери **APK**
2. Укажи путь к keystore (или создай новый через Create new)
3. Введи пароли
4. Выбери **release**
5. Нажми **Finish**

---

## Шаг 10. Установи на телефон и проверь

1. Перекинь APK на телефон (через USB, Telegram, облако)
2. Открой APK на телефоне → Установить
3. Запусти приложение
4. Перейди на страницу **Тарифы**
5. Нажми **Купить** — должна открыться шторка оплаты RuStore

### Если кнопка «Купить» не работает:

Внизу страницы тарифов есть кнопка **«Диагностика»** — нажми её и пришли скриншот. Там будет видно:
- `isAndroid: true` — приложение определено как Android
- `hasBridgeObject: true` — мост оплаты загружен
- `isAvailable: true` — RuStore SDK готов к работе
- `initError: none` — ошибок нет

---

## Частые ошибки

| Проблема | Решение |
|---|---|
| `Kotlin not resolved` | Проверь шаг 6.2 — Kotlin плагин в корневом build.gradle |
| `Cannot find symbol RuStoreBillingPlugin` | Проверь что файл скопирован в правильную папку (шаг 7) |
| `Duplicate class MainActivity` | Удали старый `MainActivity.java` (шаг 7) |
| `SDK version not found` | Android Studio → Tools → SDK Manager → установи Android 14 (API 34) |
| Шторка RuStore не открывается | Убедись что на телефоне установлен RuStore и у приложения есть продукты в RuStore Console |
| `isAvailable: false` в диагностике | На телефоне нет RuStore или нет интернета |
